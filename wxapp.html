小程序介绍

什么是小程序?

张小龙说:

“小程序是一种不需要下载、安装即可使用的应用，它出现了触手可及的梦想，用户扫一扫或者搜一下即开打开应用，也出现了用完即走的理念，用户不用关心安装太多应用的问题，应用随处可用，但又无须安装卸载。”

总结一下

  • 无需下载
  • 触手可及
  • 用完即走
  • 无须卸载

小程序特点

  • 类web，不是H5
  • 即用即走，随手可得
  • 拥有离线能力
  • 基于微信跨平台
  • 媲美原生操作体验

小程序入口

  • 扫码进入小程序
  • 搜索小程序
  • 小程序发送到桌面(Android)
  • 发送给朋友

小程序开发框架

  • JavaScript(ES6)
  • HTML/CSS

VueJS组件

视图层

视图层由 WXML 与 WXSS 编写，由组件来进行展示。

将逻辑层的数据反应成视图，同时将视图层的事件发送给逻辑层。

WXML

  • 支持数据绑定
  • 支持逻辑算术、运算
  • 支持模板、引用
  • 支持添加事件（bindtap）

WXSS

  • 支持大部分CSS特性
  • 添加尺寸单位rpx，可根据屏幕宽度自适应
  • 使用@import语句可以导入外联样式表
  • 不支持多层选择器-避免被组件内结构破坏
  • 支持基本的选择器，如：class,id,element,::before等

组件

逻辑层

逻辑层将数据进行处理后发送给视图层，同时接受视图层的事件反馈

  1. 通过App()函数来注册小程序，通过Page()函数来注册页面
  2. 提供丰富的API，既能调用微信服务，也能调用native基础功能
  3. 每个页面有独立的作用域，支持模块化开发
  4. 数据绑定，生命周期，路由管理

API

小程序开发的优点

  • 视图与逻辑层分离，通过数据驱动，不直接操作DOM
  • 全部使用https
  • 有离线能力
  • 前端组件化开发
  • 加入rpx单位，隔离设备尺寸，方便开发

一些问题

  • 仍然使用WebView渲染，非原生渲染
  • 不能在非微信环境下开发
  • 不能扩展新组件
  • 无window/document对象
  • 等等
